# Examen1
1. Es un framework creado para el desarrollo de aplicaciones móviles nativas de iOS y Android
2. Una app movil es un conjunto de componentes que funcionan de manera interna en un dispositivo gracias a una interfaz grafica. Instagram y WhatsApp
3. Tipos
    1. Nativas: aquellas aplicaciones que tienen un desarrollo 100% pensado en el sistema operativo sobre el que van a ejecutarse (calculadora, explorador de archivos, etc.)
    2. Hibridas: combinan diferentes arquitecturas pensando en que sean compatibles con mas de un sistema operativo.
    3. Web: Aplicaciones que se ejecutan directamente en el web (desde un sitio) pero nos permiten crear un acceso directo en la pantalla de nuestro celular y acceder desde ahí.
4. Componentes:
    1. View: crea divisores o secciones que nos ayudan a dividir mejor los diferentes bloques de nuestra aplicación y poder tener una buena organización general.
    2. Crea una especie de etiqueta de texto, la cual podemos editar a nuestro gusto para hacer que el texto dentro de la misma tenga el estilo que nosotros queremos.
    3. Image: Nos permite referenciar imágenes para poderlas usar en nuestra aplicación.
    4. TextInput: Funciona como un área estilizada a nuestro gusto en la que le permitimos al usuario de la aplicación ingresar texto dependiendo las restricciones que hayamos definido en el componente.
5. UseState:
    1.  Es una variable o estado que nos ayuda a generar cambios en algún componente en base a acciones realizadas por el usuario.
    2. Para definir nuevos estados a un componente ya existente.
    3. Al dar like a una historia o post de instagram, el icono de corazón cambia su apariencia y al mismo tiempo activa un contador que aumenta los likes, cuando se vuelve a dar like, este useState regresa el icono a su forma original y resta el like.
6. Lógica:
    1. Porque nos ayuda a tener una app mas dinámica e interactiva, ademas de ayudarnos a tener un mejor control de acciones o elementos en general en app que lo requieran.
    2. La app se vuelve mas estática y con menos cambios.
    3. En mi aplicación pienso usarla para poderle indicar al usuario que su stock de productos es menor al mínimo, haciendo que la barra de stock cambie de color verde (estado normal) a un amarillo (que indica la alerta de falta de stock).
